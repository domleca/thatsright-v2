# SEO for Google and LLMs — Design

**Status:** approved
**Date:** 2026-06-21
**Site:** thatsright.xyz (live, hosted on Vercel)
**Source:** `TRW/index.html`, `TRW/Privacy.html`, `Thats-Right-copy.json`, `TRW/i18n.js`

## Goals (in priority order)

1. **Discovery on Google.** Families searching things like *"voice companion Alzheimer's"* or *"app for memory loss conversations"* should find the site.
2. **Citation by LLMs.** When someone asks ChatGPT / Claude / Perplexity *"is there a voice app for Alzheimer's?"*, "That's Right" should be a name that comes up.
3. **Polished sharing.** A link pasted into iMessage / WhatsApp / LinkedIn / Slack should show a proper preview card in the right language.

## Scope summary

Packages 1 + 2 of the brainstorm. Package 3 (FAQ section, About page, Search Console submission) is explicitly deferred.

Confirmed decisions:
- Bilingual via distinct URLs: `/` (EN) and `/fr/` (FR).
- Build step: `copy.json` becomes single source of truth; a small Node script regenerates both HTML files.
- Vercel server-side redirect: `Accept-Language: fr*` hitting `/` is bounced to `/fr/`. Respects `lang-pref` cookie.
- OG image: two per-language PNGs, generated locally from SVG.
- Founder named in JSON-LD: Dom Leca.
- `applicationCategory: LifestyleApplication` (not `HealthApplication` — avoids regulated-software signal).
- `offers.availability: PreOrder` (matches the "Request access" CTA).
- Privacy page localized to French too (Dom reviews translation before ship).

## Architecture

### File layout after the change

```
TRW/
  src/                       # NEW — templates and copy
    index.template.html      # was index.html, with {{copy.hero_h1}} style slots
    privacy.template.html    # was Privacy.html, same templating
  build.js                   # NEW — Node script, ~80 lines
  i18n.js                    # unchanged runtime switcher (now also drives hard nav)
  loop.js, orb.js, site.js   # unchanged
  Thats-Right-copy.json      # SOURCE OF TRUTH for all copy
  vercel.json                # NEW — redirects + headers
  public/                    # NEW — build output, what Vercel serves
    index.html               # EN
    privacy.html             # EN
    fr/
      index.html             # FR
      privacy.html           # FR
    og-en.png                # NEW — Open Graph image, EN
    og-fr.png                # NEW — Open Graph image, FR
    robots.txt               # NEW
    sitemap.xml              # NEW — auto-generated by build.js
    llms.txt                 # NEW
```

`vercel.json` will set `"outputDirectory": "public"` so Vercel serves the build artifact, not the source.

### Build script (`build.js`)

Plain Node, **zero npm dependencies**, runs in `vercel build` or `npm run build`. A minimal `package.json` is added with a `"build": "node build.js"` script and `"engines": { "node": ">=18" }` so Vercel uses a current Node runtime. No `node_modules`.

1. Read `Thats-Right-copy.json`.
2. Read each template file.
3. For each language (en, fr):
   - Replace `{{key}}` tokens with `copy[lang][key]`.
   - Set `<html lang="...">`.
   - Inject language-appropriate `<title>`, `<meta name="description">`, `<link rel="canonical">`, `hreflang` tags, OG tags, Twitter tags, JSON-LD blocks.
   - Write to `public/` (EN at root) or `public/fr/` (FR).
4. Generate `sitemap.xml` with the four URLs.
5. Copy static assets (`*.js`, `og-*.png`, `robots.txt`, `llms.txt`) from `src/static/` to `public/`.

### Copy source of truth

Today: copy lives in three places (English baked into `index.html`, English + French in `copy.json`, French baked nowhere). Sync is manual and error-prone (documented pain in memory).

After: copy lives only in `copy.json`. Templates have `{{token}}` slots, never real text. Editing flow becomes: open `copy.json` → edit → `npm run build` → preview. One source, two outputs, zero sync drift.

For both `/` and `/fr/`:

```html
<html lang="{en|fr}">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />

  <title>{title for this language}</title>
  <meta name="description" content="{description}" />
  <meta name="robots" content="index,follow,max-image-preview:large" />

  <link rel="alternate" hreflang="en" href="https://thatsright.xyz/" />
  <link rel="alternate" hreflang="fr" href="https://thatsright.xyz/fr/" />
  <link rel="alternate" hreflang="x-default" href="https://thatsright.xyz/" />

  <!-- Open Graph -->
  <meta property="og:type" content="website" />
  <meta property="og:site_name" content="That's Right" />
  <meta property="og:title" content="{og_title for this language}" />
  <meta property="og:description" content="{description}" />
  <meta property="og:url" content="https://thatsright.xyz/{|fr/}" />
```

## Open Graph image

Two PNGs, 1200 × 630, generated locally from SVG source.

Visual rules:
- Background: cream `#faf6f0` (matches `--bg`).
- Wordmark: **"That's Right."** in Schibsted Grotesk, ~120px, ink `#2e2e2e`.
- Tagline below, ~36px, muted `#57534e`.
- Small accent orb (~80px) in `#b04a2c`, positioned with restraint.
- Thin rule between wordmark and tagline.
- No URL, no CTA, no logo soup.

Generation: write `og-en.svg` and `og-fr.svg`, convert to PNG with `rsvg-convert` (libRSVG, already on macOS via Homebrew). Dom previews both before they ship.

## Vercel configuration

`vercel.json`:
```json
{
  "buildCommand": "node build.js",
  "outputDirectory": "public",
  "cleanUrls": true,
  "redirects": [
    {
      "source": "/",
      "has": [
        { "type": "header", "key": "accept-language", "value": "^fr.*" }
      ],
      "missing": [
        { "type": "cookie", "key": "lang-pref" }
      ],
      "destination": "/fr/",
      "permanent": false
    }
  ]
}
```

`missing` clause: a user with `lang-pref` cookie already set (because they used the switcher) is exempt from the redirect.

`cleanUrls: true` so `/privacy` works without `.html`.

`permanent: false` (302) so we can adjust the rule without browser-cache lock-in.

## Language switcher behavior

`i18n.js` change:
- EN button on `/fr/...` → hard navigation to `/...`, set `lang-pref=en` cookie.
- FR button on `/...` → hard navigation to `/fr/...`, set `lang-pref=fr` cookie.
- Cookie: 1 year, `SameSite=Lax`, no `Secure` flag needed in dev but set in prod.

The current in-page text-swap behavior is removed. URLs always reflect language.

---

## Privacy page localization

`Privacy.html` becomes templated like `index.html`. The output filename drops to lowercase: `/privacy` (served as `public/privacy.html` + `public/fr/privacy.html`, with `cleanUrls: true` hiding the `.html`). The capitalized URL `/Privacy.html` gets a permanent redirect in `vercel.json` to preserve any existing inbound links.

French copy lives in `copy.json` under a new `privacy.*` namespace. Dom reviews the French legal copy before deploy. Both pages get the same `<head>` treatment, JSON-LD, and `hreflang` tags.

## Verification (before declaring done)

1. **Google Rich Results Test** — pass on both `/` and `/fr/`.
2. **OG debugger** — preview card renders correctly with right image and locale.
3. **Lighthouse SEO audit** — score, fix anything below 100.
4. **Manual `hreflang` check** — both pages reference each other.
5. **Live redirect test** — French-locale browser hitting `/` lands on `/fr/`; switcher overrides correctly via cookie.
6. **Build idempotency** — `npm run build` twice in a row produces identical output.

### Post-deploy checklist for Dom

- Verify ownership in **Google Search Console**; submit `sitemap.xml`.
- Same for **Bing Webmaster Tools** (powers ChatGPT Search).
- (Optional, ~2 weeks later) Check Search Console for crawl errors, indexed pages, queries.

## Risk notes

- **French legal translation.** Privacy page in French requires Dom's review. A sloppy auto-translation of a legal document is worse than no translation.
- **`hero_h1` flash bug.** Open layout bug (from memory) is independent of this work, but the templating refactor might shift its symptoms. Will note any regressions and surface them — not in scope to fix here.
- **Existing hand-edited bundled files.** The numerous `Thats-Right-*.html` design checkpoints in `TRW/` are untouched. They are reference snapshots, not served by Vercel.
